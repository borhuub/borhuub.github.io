Administrator

Writeups sobre la máquina Administrator en Active Directory de dificultad Media. Enumerando con bloodhound encontraremos un path claro que seguir para ir moviendonos entre usuarios, llegado cierto momento obtendremos un archivo con credenciales a través de FTP con el cual obtendremos el usuario que contiene la user flag. Para completar la escalada de privilegios volveremos a abusar de permisos GenericAll para realizar un kerberoast attack sobre el usuario ethan, luego abusando de permisos GetChangesAll y GetChanges realizaremos un DCSync attack para convertirnos en domain admin.

ForceChangePassword - Net rpc#

Si volvemos a comprobar el bloodhound vemos que michael tiene el permiso ForceChangePassword sobre el usuario Benjamin, es decir, tenemos la capacidad de cambiarle la contraseña.

Lanzamos el comando con net rpc y cambiamos la contraseña a Benjamin a “password”.

❯ net rpc password "benjamin" 'password' -U ADMINISTRATOR.HTB/michael%password -S "10.10.11.42"

Mirando el bloodhound de benjamin, solo vemos que es miembro del grupo Share Moderators. Esto me puede dar una pista que puede tener permiso en algún tipo de carpeta compartida. DCSync Attack - nxc#

https://books.spartan-cybersec.com/cpad/persistencia-y-post-explotacion-en-ad/dcsync

Vemos que el usuario ethan del cual tenemos control actualmente tiene los permisos GetChangesAll y GetChanges sobre el dominio ADMINISTRATOR.HTB.

Estos permisos nos permiten con nxc ejecutar el ataque DCSync, que consiste en solicitar información sensible del domain controler haciéndonos pasar por otro “domain controler”.
