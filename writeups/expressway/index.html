Acceso inicial#

IKE Aggressive Mode#

Entonces ahora, voy a usar ike-scan que es una herramienta típica para hacer fingerprint a puertos IPsec. Junto con la opción –agressive para tratar de que nos identifique algo de información útil con la que seguir.

sudo ./ike-scan --agressive 10.129.17.85

El escaneo agressivo, nos da mucha información valiosa con la que poder continuar. Como por ejemplo mostrándonos el identificador del usuario ike@expressway.htb y además del parámetro Hash(20 bytes) el cual nos indica que el servidor envía un hash derivado de PSK en claro que podemos intentar crackear offline.

sudo ./ike-scan 10.129.17.85 -M -A --id=ike@expressway.htb -pskcrack=expressway.hash

PSK Cracking#

Una vez obtenido el derivado del PSK, pasamos el hash por ikescan2john para crear el hash con el cual podemos obtener la contraseña.

ikescan2john expressway.htb

Ya ahora con el hash crackeable obtenemos la contraseña usando John.

john --format=ike --wordlist=/usr/share/SecLists/Password/Leaked-Databases/rockyou.txt ike_hash

ike : freakingrockstarontheroad

Ahora con la contraseña en nuestro poder, probamos a acceder a través de shh y obtenemos la user flag.

Escalada de privilegios#

Primero empiezo enumerando los permisos existentes en el sistema. Pruebo un sudo -l pero veo que no tengo permisos.

Entonces me pongo a mirar los permisos del usuario ike y veo que es miembro del grupo proxy aunque luego de enumerar los permisos no encuentro nada interesante.

Entonces me pongo a mirar los puertos expuestos en la máquina y encuentro que la máquina tiene el puerto SMTP (25) expuesto.

Con lo cual, me pongo a enumerar-lo y veo que la versión del servicio que esta corriendo es Exim4 4.98.2. Me pongo a buscar vulnerabilidades conocidas pero para esta versión las vulnerabilidades ya habían sido parcheadas.

Chroot Explotation - CVE-2025-32463#

Tras la búsqueda de la versión en internet, descubrimos que esta versión contiene una vulnerabilidad crítica que nos permite escalar privilegios.

https://www.incibe.es/index.php/incibe-cert/alerta-temprana/vulnerabilidades/cve-2025-32463

Esta vulnerabilidad permite a un usuario local ganar acceso como root mediante la opción --chroot (R).

Para poder llevar a cabo la explotación vamos a crear un directorio temporal llamado pwn para construir un entorno pequeño de chroot.

Después procedemos a crear dos carpetas que simularan el entorno chroot.

  • libnss_ aquí colocaremos la biblioteca compartida que imita un módulo NSS.
  • pwn/etc aquí colocaremos el archivo nsswitch.conf que fuerza a utilizar el módulo del directorio que controlamos.

Ahora crearemos el archivo nsswitch.conf que forzará a consultar nuestro modulo malicioso. Y también creamos el script en C que luego compilaremos.

echo "passwd: /pwn" > pwn/etc/nsswitch.con
export TERM=xterm #Para poder usar nano
nano pwn.c

En este modulo malicioso lo que hacemos es asigar-nos permisos root UID y GID para luego ejecutar una shell con los permisos asignados.

Y ahora compilamos el script en C para crear un modulo malicioso que se ejecutará cuando ejecutemos sudo chroot.

gcc -shared -fPIC -Wl,-init,pwn -o libnss_/pwn.so.2 pwn.c

Ahora hacemos que usando sudo -R que cambie el directorio root a pwn, para que luego ejecute pwn haciendo que nos cargue los módulos maliciosos.

sudo -R pwn pwn

Al ejecutar la operación y hacer un id vemos que se nos a asignado los permisos UID y GID de root confirmando que hemos conseguido elevar nuestros privilegios.
